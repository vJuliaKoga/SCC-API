## 0. 前提条件

以下を事前にインストールしておくこと。

- Docker / Docker Compose
- [Java 21](https://www.oracle.com/jp/java/technologies/downloads/#jdk21-windows)
- [Gradle](https://gradle.org/releases/)
  - ※Wrapper使用時は不要だが初回生成時に必要

### Java / Gradle 環境変数設定（Windows）

環境変数を設定する。

- `JAVA_HOME`

  ```
  C:\Program Files\Java\jdk-21.0.10
  ```

- `GRADLE_HOME`

  ```
  C:\gradle\gradle-9.4.1
  ```

Path に追加

```
%JAVA_HOME%\bin
%GRADLE_HOME%\bin
```

設定後は **PowerShell を再起動** すること。

## 1. Observability 起動

プロジェクトルートで実行。

```shell
docker compose -f observability/docker-compose.yml up -d
```

### 確認

- Jaeger
  [http://localhost:16686](http://localhost:16686)

- Prometheus
  [http://localhost:19090](http://localhost:19090)

---

## 2. gRPC Backend 起動

```shell
cd grpc-backend
gradle wrapper   # 初回のみ
.\gradlew.bat bootRun
```

### 起動確認（別ターミナル）

```shell
Invoke-RestMethod http://localhost:19091/actuator/health
netstat -ano | findstr :19091
netstat -ano | findstr :29090
```

#### 期待結果

- `/actuator/health` が `UP`
- ポート
  - `19091`（HTTP）
  - `29090`（gRPC）

---

Invoke-RestMethod -Method POST http://localhost:19092/api/orders `

## 6. gRPC モード切替

### 設定変更

`bff/src/main/resources/application.yml`

```yaml
app:
  call-mode: grpc
```

### 再起動

```shell
.\gradlew.bat bootRun
```

もしくは

```shell
.\gradlew.bat bootRun --args="--app.call-mode=grpc"
```

### よくある問題

#### Gradle コマンドが動かない

- PowerShell 再起動していない
- 環境変数未設定

#### API が返らない

- backend 未起動
- ポート競合
- BFF のモード設定ミス

#### gRPC に切り替わらない

- BFF 再起動していない
- application.yml の保存忘れ

## 10. 補足

- Gradle は基本 `gradlew` を使用する（環境依存を避けるため）
- Windows の場合は `.bat` を使用すること
- 各サービスはログに `Started` が出るまで待つこと
